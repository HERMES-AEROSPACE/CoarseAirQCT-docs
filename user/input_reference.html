Input File Reference

CoarseAIR is driven by three plain-text input files living in the input/ subfolder of the run directory:

  • CoarseAIR.inp — the main input file. Everything below is about this file.

  • GenerateLevels.inp — level-generation parameters (first-time runs for a new molecule only).

  • PlotPES.inp — PES grid for the coarseair-plotpes.x visualiser.

This page enumerates every section of CoarseAIR.inp and links each line to the code that consumes it. The authoritative parser is src/Input_Class.F90 (~2400 lines); see it for corner cases.

Note

File format rules that apply to every input block:

  • The order of lines does not matter, as long as sizing lines (e.g. Nb of Species = N) precede the lines that depend on them.

  • Indentation does not matter.

  • Whitespace around the = does matter. System = O3 works; System=O3 does not.

Top-level pipeline flags

Generating Energy Levels?        = yes | no
Preprocessing Energy Levels List? = yes | no
Running Trajectories?             = yes | no
Postprocessing Trajectories?      = yes | no

These four flags are consumed by CoarseAIR.sh to decide which stages of the pipeline to execute. See Running Trajectories.

System and atomic-composition block

System               = O3             ! name of the chemical system
Nb of Species        = 2
  Species 1          = O2             ! target
  Species 2          = O              ! projectile
Nb of Atoms          = 3
  Atom 1             = O
  Mass [a.u.] of Atom 1 = 29148.94559   ! (example)
  Atom 2             = O
  Mass [a.u.] of Atom 2 = 29148.94559
  Atom 3             = O
  Mass [a.u.] of Atom 3 = 29148.94559
Nb of Molecules      = 1              ! 1 for O3, 3 for CHN, etc.
  Molecule 1         = O2
  Nb of Atoms, Molecule 1 = 2
Database Path        = /path/to/coarseair/dtb
Nb of Initial Molecules = 1           ! 1 for 3-atom, 2 for 4-atom

For isotope-labelled systems, set the atomic mass explicitly here; the PES is unchanged but reduced masses differ.

Translational energy / temperature block

The single line Relative Translational Energy Model = X selects one of four relative-velocity sampling strategies. The follow-up lines depend on X:

Model

What it does

Required follow-up lines

FixedRelE

Every trajectory starts with the same relative translational energy \(E_{\text{rel}}\). One cross- section is produced per energy; no averaging over \(E\) is applied.

Nb of Translational Energies = N + Translational Energy [Eh] k for \(k = 1\dots N\)

Boltzmann

Energies are drawn from the Maxwell–Boltzmann translational distribution at temperature \(T\) (inverse-CDF sampling of \(\hat E\,e^{-\hat E}\) — see Sampling). Thermal rate coefficients \(k(T)\) are then an arithmetic mean of \(\sigma(\hat E)\).

Nb of Translational Temperatures = N + Translational Temperature [K] k for \(k = 1\dots N\)

Gaussian

Energies are drawn from a Gaussian of user-defined mean and standard deviation. Useful for narrow-bandwidth laser-excitation studies or for reproducing non-thermal distributions from a CFD cell.

Average Translational Energy [Eh] = <mean> + Translational Energy SD      [Eh] = <sigma>

FixedTotEn

Every trajectory starts with the same total energy (translational + internal), allowing the vibrational population to vary with the \((v, J)\) choice while \(E_{\text{tot}}\) is held fixed. Used mainly for diagnostic studies of vibration-specific thresholds.

Fixed Total Energy [Eh] = <E_tot>

The Nb of Internal Temperatures block is separate — it sets the internal (ro-vibrational) temperature used by the binning and by Boltzmann-weighted initial-level selection; it is independent of the translational model:

Nb of Internal Temperatures  = 1
  Internal Temperature [K] 1 = 10000

Worked examples of each translational model:

! ---  FixedRelE : three fixed collision energies  ---
Relative Translational Energy Model = FixedRelE
Nb of Translational Energies        = 3
  Translational Energy [Eh] 1 = 0.025      ! ~ 0.68 eV
  Translational Energy [Eh] 2 = 0.10       ! ~ 2.72 eV
  Translational Energy [Eh] 3 = 0.40       ! ~ 10.9 eV

! ---  Boltzmann : Maxwell thermal averaging at three temperatures  ---
Relative Translational Energy Model = Boltzmann
Nb of Translational Temperatures    = 3
  Translational Temperature [K] 1 = 5000.d0
  Translational Temperature [K] 2 = 10000.d0
  Translational Temperature [K] 3 = 20000.d0

! ---  Gaussian : narrow distribution around 0.1 Eh  ---
Relative Translational Energy Model   = Gaussian
Average Translational Energy [Eh]     = 0.10d0
Translational Energy SD      [Eh]     = 0.01d0

! ---  FixedTotEn : fixed total energy  ---
Relative Translational Energy Model = FixedTotEn
Fixed Total Energy [Eh]             = 0.25d0

Choosing between the models:

  • If you want direct cross-sections \(\sigma(E)\) on a controlled energy grid, use FixedRelE.

  • If you want thermal rate coefficients \(k(T)\) in one shot, use Boltzmann.

  • If you are reproducing a non-thermal distribution from upstream simulation data, use Gaussian (or combine a vector of FixedRelE points with post-processing reweighting).

  • If you need to hold the total energy fixed and vary the partition between translation and internal modes, use FixedTotEn.

See Sampling for the Monte Carlo derivations that connect each model to its Maxwellian / Gaussian / delta-function integrand.

Impact-parameter block

The Initial Impact Parameter Model line picks one of the four sampling strategies described in Sampling. Each model expects a different sub-block immediately after. Sign convention: CoarseAIR reads impact parameters as negative numbers — the magnitude is what matters, the sign is an internal flag.

Normal — uniform sampling in \(b^2\) from 0 to a single \(b_{\max}\):

Initial Impact Parameter Model = Normal
  Initial Impact Parameter = -14.0d0     ! b_max in bohr (magnitude)

Random — uniform sampling in \(b\) (note the different weighting — requires Jacobian correction during rate extraction):

Initial Impact Parameter Model = Random
  Initial Impact Parameter = -14.0d0

Stratified — divides the impact-parameter range into N strata, each with its own trajectory budget. Produces much tighter cross-section error bars at fixed cost. The Strata Type is either Circles (nested discs of radius \(b_i\) with cumulative counts) or Rings (disjoint annuli between successive \(b_i\) with per-annulus counts):

Initial Impact Parameter Model = Stratified
  Strata Type = Circles
  Nb of Stratified Sampling Ranges = 7
    Nb of Trajectories in Range 1 = 1
    Impact Parameter for Range 1 = -0.25d0
    Nb of Trajectories in Range 2 = 1
    Impact Parameter for Range 2 = -0.5d0
    Nb of Trajectories in Range 3 = 1
    Impact Parameter for Range 3 = -1.0d0
    Nb of Trajectories in Range 4 = 1
    Impact Parameter for Range 4 = -3.0d0
    Nb of Trajectories in Range 5 = 1
    Impact Parameter for Range 5 = -5.0d0
    Nb of Trajectories in Range 6 = 1
    Impact Parameter for Range 6 = -8.0d0
    Nb of Trajectories in Range 7 = 1
    Impact Parameter for Range 7 = -14.0d0

Rings example (equivalent physics, different weighting of the allocation):

Initial Impact Parameter Model = Stratified
  Strata Type = Rings
  Nb of Stratified Sampling Ranges = 2
    Nb of Trajectories in Range 1 = 1
    Impact Parameter for Range 1 = -6.0d0
    Nb of Trajectories in Range 2 = 99
    Impact Parameter for Range 2 = -14.0d0

FixedAngMom — selects \(b\) so that every trajectory has a prescribed total orbital angular momentum, useful for reactive- scattering analyses at fixed \(\ell\):

Initial Impact Parameter Model = FixedAngMom
  Total Angular Momentum = 80.0d0       ! in atomic units

Integration-control block:

Backward Integrating?          = no
Initial time step              = 1.0             ! a.u.

Process-index encoding (output)

The rate-coefficient files encode each reactive process with a single integer d that is constructed from the final arrangement (which pair of atoms is bonded) and the final-state class (bound / quasi-bound / dissociating / predissociating).

For a 3-atom system, each trajectory contributes to one of four inelastic/exchange channels:

Channel

Process index d

Inelastic: \(AB_i + C \to AB_j \equiv C\)

\(d = 1 + d_1\)

Exchange to 2nd pair: \(AB_i + C \to AC_k \equiv B\)

\(d = 1 + (\mathcal{N}_1^G + 1) + d_2\)

Exchange to 3rd pair: \(AB_i + C \to BC_l \equiv A\)

\(d = 1 + (\mathcal{N}_1^G + 1) + (\mathcal{N}_2^G + 1) + d_3\)

Here \(\mathcal{N}_i^G\) is the number of ro-vibrational levels / groups in the \(i\)-th pair’s molecule, and \(d_m\) is the final-level index within that pair (with \(d_m = 1\) reserved for dissociation from the mol-th final molecule).

For 4-atom systems each trajectory ends with two product diatoms, and the d index runs over all pair–pair combinations. Writing \(\mathcal{N}_k\) for the number of levels/groups in the \(k\)-th atom pair of the 4-atom enumeration (1=AB, 2=AC, 3=AD, 4=BC, 5=BD, 6=CD), and \(d_k = 1\) for the “pair-\(k\) dissociated” case and \(d_k = s+1\) for level \(s\) otherwise, the 4-atom formulas are:

\[\begin{split}&\text{Inelastic } AB_i + CD_j \to AB_k + CD_l : \\ &\qquad d \;=\; 1 + (d_1 - 1)(\mathcal{N}_6^G + 1) + d_6,\end{split}\]
\[\begin{split}&\text{Exchange to 2nd pair } AB_i + CD_j \to AC_m + BD_n : \\ &\qquad d \;=\; 1 + (\mathcal{N}_1 + 1)(\mathcal{N}_6 + 1) + (d_2 - 1)(\mathcal{N}_5^G + 1) + d_5,\end{split}\]
\[\begin{split}&\text{Exchange to 3rd pair } AB_i + CD_j \to AD_p + BC_q : \\ &\qquad d \;=\; 1 + (\mathcal{N}_1 + 1)(\mathcal{N}_6 + 1) + (\mathcal{N}_2 + 1)(\mathcal{N}_5 + 1) + (d_3 - 1)(\mathcal{N}_4^G + 1) + d_4.\end{split}\]

\(d = 1\) is reserved (retained only for compatibility with old file formats).

Final arrangements in trajectory output are encoded as 16 * m + n, where \(m\) is the final-diatom pair index (3-atom: 1=AB, 2=AC, 3=BC; 4-atom: 1-6 per the convention above) and \(n \in \{0, 1, 2, 3\}\) encodes bound / quasi-bound / dissociating / pre-dissociating. See Post-processing for the complete arr_f mapping tables.

Worked example — NO + C (3-atom)

For the NO + C input (input/CNO/UCI/NO+C/input/CoarseAIR.inp) the atom order in the input block is Atom 1 = N, Atom 2 = O, Atom 3 = C, so CoarseAIR’s pair numbering becomes

Pair

Atoms

CoarseAIR “Molecule”

Physical product

1

N–O

Molecule 1 = NO

Incoming arrangement (inelastic / elastic).

2

N–C

Molecule 2 = CN

Exchange: C-atom captured by N.

3

O–C

Molecule 3 = CO

Exchange: C-atom captured by O.

All three atoms are chemically distinct, so the three channels are genuinely different reactions (no identical-atom averaging applies), and each molecule has its own rovib ladder: \(\mathcal{N}^G_{\rm NO} \ne \mathcal{N}^G_{\rm CN} \ne \mathcal{N}^G_{\rm CO}\).

Case A — Thermal / coarse-grained mode (Sorting Method = State-Specific with all levels aggregated into a single bin per molecule, so \(\mathcal{N}^G_{\rm NO} = \mathcal{N}^G_{\rm CN} = \mathcal{N}^G_{\rm CO} = 1\).) One rate coefficient per channel:

d

Channel

Meaning

2

Inelastic (pair 1)

Pair-1 dissociation: NO + C → N + O + C.

3

Inelastic (pair 1)

NO + C → NO(bound) + C (inelastic / elastic).

4

Exchange pair 2 (CN)

Pair-2 dissociation: NO + C → N + O + C (via CN channel).

5

Exchange pair 2 (CN)

NO + C → CN + O.

6

Exchange pair 3 (CO)

Pair-3 dissociation: NO + C → N + O + C (via CO channel).

7

Exchange pair 3 (CO)

NO + C → CO + N.

Total: d = 2 … 7 (6 channels).

Case B — State-to-state mode (Sorting Method = State-Specific with every ro-vibrational level as its own bin.) Representative level counts on the UCI diatomic potentials: \(\mathcal{N}^G_{\rm NO} \approx 1000\), \(\mathcal{N}^G_{\rm CN} \approx 900\), \(\mathcal{N}^G_{\rm CO} \approx 6000\). Applying the 3-atom formulas (with \(\mathcal{N}^G + 1\) as each block stride):

d interval

Channel

Meaning

d = 2

Inelastic (pair 1, NO)

Pair-1 dissociation → N + O + C.

d ∈ [3, 1002]

Inelastic (pair 1, NO)

NO(v’, J’) + C for each of the ~1000 final NO levels (\(d_1 = 2 \dots 1001\)).

d = 1003

Exchange pair 2 (CN)

Pair-2 dissociation via the CN channel.

d ∈ [1004, 1903]

Exchange pair 2 (CN)

CN(v’, J’) + O for each of the ~900 final CN levels (\(d_2 = 2 \dots 901\)).

d = 1904

Exchange pair 3 (CO)

Pair-3 dissociation via the CO channel.

d ∈ [1905, 7904]

Exchange pair 3 (CO)

CO(v’, J’) + N for each of the ~6000 final CO levels (\(d_3 = 2 \dots 6001\)).

Total: d = 2 … 7904 (~7900 channels per initial NO level). The state-to-state rate table therefore contains roughly \(7900 \times 1000 \approx 7.9 \times 10^{6}\) entries across all initial levels — an order of magnitude larger than for a symmetric system like H + H2 (~1000 channels per initial level). The large CO range dominates because the CO diatomic has the deepest well and supports the most bound ro-vibrational states.

In the CNO system the three arrangements are not interchangeable (N, C, O are distinct), so Distinguish Exchanges between Each Other? = yes is the normal setting: each of the three blocks is reported as its own physical channel (inelastic vs CN exchange vs CO exchange) rather than summed.

Worked example — H2 + H2 (4-atom)

Four identical hydrogens, six atom-pair partitions, two product diatoms per trajectory. Because all atoms are identical, the six pair-level counts are all equal: \(\mathcal{N}^G_1 = \dots = \mathcal{N}^G_6 \equiv \mathcal{N}^G\).

Case A — Thermal / coarse-grained mode (one bin per molecule, \(\mathcal{N}^G = 1\)).

With a single group per pair, \(\mathcal{N}^G + 1 = 2\), and the inelastic formula \(d = 1 + (d_1 - 1)(\mathcal{N}^G_6 + 1) + d_6\) reduces to \(d = 1 + 2(d_1 - 1) + d_6\) with \(d_1, d_6 \in \{1, 2\}\) (1 = dissociated, 2 = bound):

d interval

Channel

Meaning

d ∈ [2, 5]

Inelastic (pair 1 + pair 6)

AB×CD partition: 4 combinations of (AB, CD) each being bound or dissociated. d = 2: both dissociated (4-atom atomisation). d = 3, 4: one bound, one dissociated (single dissoc.). d = 5: both bound (pure inelastic / elastic).

d ∈ [6, 9]

Exchange pair 2 (pair 2 + pair 5)

AC×BD partition — 4 combinations as above.

d ∈ [10, 13]

Exchange pair 3 (pair 3 + pair 4)

AD×BC partition — 4 combinations as above.

Total: d = 2 … 13 (12 channels).

Case B — State-to-state mode, \(\mathcal{N}^G = 345\) (the BKMP2 H2 rovib ladder).

Here \(\mathcal{N}^G + 1 = 346\), and the inelastic formula \(d = 1 + 346\,(d_1 - 1) + d_6\) indexes a \(346 \times 346\) grid of (pair-1 outcome, pair-6 outcome) combinations:

d interval

Channel (and dominant sub-ranges)

Meaning

d ∈ [2, 119 717]

Inelastic (AB × CD)

346² combinations. Per-\(d_1\) sub-block:

  • d ∈ [2, 346]: pair 1 dissociated, pair 6 ∈ {diss + 345 levels}.

  • d ∈ [347, 692]: pair 1 in level 1, pair 6 varies.

  • …

  • d ∈ [119 372, 119 717]: pair 1 in level 345, pair 6 varies.

d ∈ [119 718, 239 433]

Exchange pair 2 (AC × BD)

346² combinations, indexed by \((d_2, d_5)\).

d ∈ [239 434, 359 149]

Exchange pair 3 (AD × BC)

346² combinations, indexed by \((d_3, d_4)\).

Total: d = 2 … 359 149 (~360 000 channels per initial state) × ~120 000 initial \((v_1, J_1, v_2, J_2)\) states. The full 4-atom StS rate table therefore contains \(\sim 4 \times 10^{10}\) entries — which is why 4-atom production runs are almost always coarse-grained (Sorting Method = RoVib-CG with \(\mathcal{N}^G \sim 50\)–100 bins per pair) rather than truly state-to-state. Proc_<d>.csv files are emitted only for non-zero channels and are compressed to binary when Writing Rate Files in Binary Format? = yes.

In identical-atom post-processing (Identical Diatoms? = yes and Distinguish Exchanges between Each Other? = no), the three 346²-sized arrangement blocks collapse into a single block — the physical rate for each (initial, final) pair of H2 levels summed over pair-swap symmetries.

Decoding d by hand vs with the tooling

In practice the user rarely inverts \(d\) by hand: the Python and MATLAB post-processing tools (see Post-processing) do the mapping automatically and report human-readable channel labels (NO + C -> CN + O, v' = 12, J' = 35). The formulas above are useful for sanity-checking an ad-hoc grep through a Proc_*.csv file, or for writing a custom rate-database loader.

Secondary input files

GenerateLevels.inp — consumed only when Generating Energy Levels? = yes. Controls the \((v, J)\) grid boundaries, the radial grid for the diatomic Schrödinger solver, and the output file name. Refer to the comments inside the template file.

ProcessesToRunList.inp — consumed when Starting Level, Molecule 1 = 0 (and Starting Level, Molecule 2 = 0 for 4-atom systems). This is the mechanism for running an arbitrary hand-picked subset of initial levels rather than a contiguous [Starting … Final] range. The file lives next to CoarseAIR.inp in the same input/ directory.

Format for 3-atom systems. One integer per line, each the level index of the target diatom:

# ProcessesToRunList.inp  (3-atom)
1
1000
2000
3000

That run would exercise only four initial states — levels 1, 1000, 2000, and 3000 of Molecule 1. Activate it by setting:

Sorting Method, Molecule 1 = State-Specific
  Starting Level, Molecule 1 = 0     ! <- triggers list-file mode
  Final    Level, Molecule 1 = 0

Format for 4-atom systems. One pair per line, written i-j, where i is the level of Molecule 1 and j is the level of Molecule 2:

# ProcessesToRunList.inp  (4-atom)
1-1
10-20
20-30
30-40
40-50
50-54

Activate by setting Starting Level = 0 on both initial molecules:

Sorting Method, Molecule 1 = State-Specific
  Starting Level, Molecule 1 = 0
  Final    Level, Molecule 1 = 0
Sorting Method, Molecule 2 = State-Specific
  Starting Level, Molecule 2 = 0
  Final    Level, Molecule 2 = 0

Typical uses:

  • Active-learning campaigns that iteratively pick low-statistics or high-uncertainty levels to revisit (scripts/launching/CoarseAIR_ActiveLearning.sh).

  • Restart after a truncated run: re-run only the levels that were not yet completed.

  • Targeted cross-validation across PES variants on a small representative set of initial states.

Examples of ProcessesToRunList.inp ship with every system under input/<System>/<PES>/.../input/ — e.g. input/CNO/UCI/NO+C/input/ProcessesToRunList.inp (3-atom format) and input/NaNbNcNd/NASA/input/ProcessesToRunList.inp (4-atom i-j format).

PlotPES.inp — consumed by coarseair-plotpes.x. Controls the grid of atomic configurations at which the PES is sampled for visualisation. Whether a diatomic or triatomic surface is plotted is selected via the line PES or Diatomic = PES.
