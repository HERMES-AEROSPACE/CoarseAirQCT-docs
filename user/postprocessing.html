Post-processing

Once trajectories have been integrated, the post-processing stage converts raw outcomes into cross-sections and rate coefficients. This page describes the output file layout, the two built-in statistics stages, and the Python / MATLAB analysis tooling shipped alongside the code.

The underlying estimator (Bernoulli cross-section with binomial-variance standard deviation) is derived in Rate & Cross-Section Extraction.

Pipeline overview

Two successive executables consume trajectory output:

  1. coarseair-trajectoriesstats.x — reads trajectory files, accumulates per-channel cross-sections into statistics.csv.

  2. coarseair-posttrajectories.x — reads statistics.csv and energy grids, produces the Proc*.csv rate files.

Both are invoked automatically by CoarseAIR.sh when Postprocessing Trajectories? = yes in the input.

Arrangement codes (arr_i / arr_f)

Every arrangement code written to trajectories.csv or statistics.csv — both the initial state arr_i and the final state arr_f — is encoded as

\[\texttt{arr} \;=\; 16\,m \;+\; n,\]

where \(m\) is the atom-pair index (using CoarseAIR’s standard pair numbering — see Quasi-Classical Trajectories) and \(n\) is the final-state class of that diatom, with the four legal values

\(n\)

Label

Meaning

0

Bound

Stable ro-vibrational level below the centrifugal-barrier maximum (and below the dissociation asymptote). The normal inelastic / exchange case.

1

Quasi-bound

Classically trapped behind the centrifugal barrier but above the asymptote \(V_\infty\). Metastable; will eventually tunnel out in a full quantum treatment.

2

Dissociating

Classically unbound — atoms separated beyond the barrier at trajectory termination. This diatom’s pair will not re-form a bound state.

3

Pre-dissociating

Bound by the classifier at trajectory termination but marked at risk of dissociation during post-termination analysis (short-lifetime quasi-bound, energy close to \(V_{\max}(J)\)). Used sparingly.

See Arrangement Classification for the classifier decision tree that assigns \(n\).

The 4-atom output therefore gives up to 24 possible values for each of the two arr_f columns (six pair indices × four state classes). Four-atom dissociation and single-dissociation outcomes (only one diatom survives, with one or two free atoms) follow from the combination of the two arr_f values: if one is of class \(n = 2\) while the other is \(n = 0\), the outcome is a single dissociation; if both are \(n = 2\), the outcome is four-atom atomisation.

Note

The \(16\,m + n\) encoding lets the code reserve up to \(m = 0\) for future extensions (5-atom systems, for instance); it does not mean the arr column starts at 0. The smallest value used in practice is 16 (pair 1, bound).

Reverse-engineering the code from an arr value is a divmod: \(m = \lfloor \texttt{arr}/16 \rfloor\), \(n = \texttt{arr} \bmod 16\).

Proc*.csv — rate coefficients

Location:

Text/<System>/Rates/T_<TTran>_<TInt>/Proc_<iProc>.csv

where iProc identifies the initial state (iProc = i for 3-atom systems, or iProc = (i - 1) * N_Bins^B + j for 4-atom systems).

Header block (top of the file) includes metadata:

  • System: — the chemical system.

  • PES: — the surface used, or Merged if multiple PES are summed.

  • In. Molecules: — reactants.

  • No Processes: — number of process rows below.

  • No Trajectories: — trajectories summed into this file.

  • In. Lev.s/Bins: — initial level/bin indices.

  • Part. Func.s Ratios: — contribution of these initial states to the molecule’s partition function.

  • Tran Temperature: / Int Temperature: — temperatures used.

Data rows:

Column

Symbol

Units

1

Process index d

(integer)

2

\(k_{i,j}\) or \(k_{ij,kl}\)

cm³/s

3

Standard deviation \(k^{SD}\)

cm³/s

The d encoding splits inelastic / exchange / dissociation channels and is documented in Input File Reference.

Controlling the aggregation

Seven CoarseAIR.inp flags control how raw trajectory outcomes are aggregated into statistics:

  • Distinguish between PESs? — if yes, produce one statistics file per electronic surface (plus a merged one).

  • Preserving Even / Oddness of Condition 1? — apply nuclear-spin selection rules (relevant for homonuclear diatoms).

  • Identical Diatoms? — for 4-atom systems, symmetrise over equivalent diatoms (e.g. N2 + N2). Both diatoms must be treated identically (same binning).

  • Distinguish Exchanges between Each Other? — for identical-atom systems, treat different permutations as separate exchange reactions (labelled mode) vs. sum them (indistinguishable mode).

  • Distinguish Exchanges from Inelastic? — treat same-element atom exchange as exchange (rather than inelastic).

  • Writing Rate Files in Binary Format? — binary output for speed.

  • Temperature for Distribution Function Computation [K] — extra equilibrium distribution for diagnostics.

Because none of these flags change the trajectory output, they can be toggled between post-processing passes without re-running the expensive trajectory stage.

PyCoarseAIR

For downstream analysis, Simone Venturi maintains PyCoarseAIR — a Python tool that reads raw CoarseAIR output and assembles it into kinetic databases compatible with the PLATO master-equation solver.

Workflow:

  1. Download the chemistry system’s HDF5 database from https://www.chess.aerospace.illinois.edu/databases.

  2. Install dependencies: pip3 install numpy pandas matplotlib h5py.

  3. Edit PyCoarseAIRFldr/exec/PyCoarseAir.py to set the local paths (PyCoarseAIRFldr, DtbHDF5Fldr, DtbWriteFldr, OutputWriteFldr).

  4. Run:

    cd PyCoarseAIRFldr/exec
python3 ./PyCoarseAir.py ../input/<System>/

Outputs:

  • PyCoarseAIRFldr/kinetic/<System>/ — rate coefficients in PLATO format.

  • PyCoarseAIRFldr/thermo/<System>/ — level / group distribution functions at the configured initial temperature.

Both PLATO (CFD) and dmsAIR (DSMC) consume these files directly — see About CoarseAIR for the HERMES-Aerospace tool stack.

MCoarseAIR

extra/MCoarseAIR/ contains a MATLAB suite for plotting and statistical analysis of CoarseAIR output — PES visualisation, vibrational-ladder plots, Arrhenius fits, and Bayesian-NN uncertainty studies. Add MCoarseAIR/ to the MATLAB path and open RunAnalysis.m as the entry point.

Diagnostic checks

Before trusting a run, inspect:

  • Energy conservation per trajectory. See Trajectory Integrators.

  • Sum rule: cross-sections for all outgoing channels from one starting level should sum to \(\pi b_{\max}^2\). See Rate & Cross-Section Extraction.

  • Detailed balance between forward and reverse rates at the same temperature — within statistical error. Persistent violations flag classification bugs or zero-point-energy artefacts.

All three checks are available in the MATLAB and Python post-processing tools.
