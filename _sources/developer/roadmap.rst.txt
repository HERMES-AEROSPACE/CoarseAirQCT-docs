Development Roadmap
===================

This page tracks in-progress work, short-term goals, and longer-term
directions for CoarseAIR and its documentation. Items are not strict
commitments — treat this as guidance for contributors looking for a
starting point.


In progress
-----------

Items currently being worked on by the CoarseAIR team.

- **CoarseAIR+ (surface-hopping fork).** Fewest-switches surface
  hopping (FSSH) on the 9 O\ :sub:`3` electronic surfaces. Initial
  implementation complete on disk under
  ``~/WORKSPACE/CoarseAIR/coarseair+``. Outstanding items: validation
  against Tully (1990) reference models (done as a stand-alone Python
  clone; production-code validation in flight), deployment as a
  sibling CoarseAIR+ documentation site.

- **Documentation — this site.** Continually expanding the reference
  pages. Known gaps: worked examples for CNO / CO\ :sub:`2` /
  NO\ :sub:`2` kinetics, a tutorial for stratified impact-parameter
  sampling, and integrator benchmarks.


Short-term goals (next 1–3 months)
----------------------------------

Items worth picking up by any contributor.

**Testing and CI.**

- Port the manual ``CompareToVVTC_N3.m`` regression into an automated
  Python test that runs on every push to ``Clement-Adiabatic``.
- Add an H\ :sub:`3` smoke test to GitHub Actions (build +
  10-minute run + rate comparison to a checked-in reference).
- Add a multi-surface dispatch test on O\ :sub:`3` that asserts
  ``iPES`` distribution matches input degeneracies.
- Drive the full validation checklist from
  :doc:`regression_tests`.

**Documentation.**

- Write worked examples for CO\ :sub:`2`, NO\ :sub:`2`, CNO in
  ``examples/`` — current stubs describe kinetics but lack a
  step-by-step run walkthrough.
- Add integrator benchmark page (Bulirsch–Stoer vs alternative
  symplectic / higher-order schemes on H\ :sub:`3` and N\ :sub:`3`).
- Page on Gaussian / histogram / WKB binning variants and their
  impact on zero-point-energy artefacts in QCT.

**Quality of life.**

- Clean up the ``i_Debug = .True.`` flag system so debug output can
  be toggled at runtime, not at compile time.
- Consolidate the launcher scripts
  (``scripts/launching/*-UIUC.{pbs,slurm}``,
  ``*-Pleiades.pbs``) — the per-site copies have drifted.
- Port ``extra/MCoarseAIR/`` analyses to a Python library so
  post-processing doesn't require MATLAB.


Medium-term goals (3–12 months)
-------------------------------

- **CoarseAIR+ production readiness.** Validate FSSH rates for
  O\ :sub:`3` against literature; publish the initial paper; extend
  to CO\ :sub:`2`, NO\ :sub:`2`, CNO, N\ :sub:`2`\ O (which have
  documented multi-surface manifolds).
- **Alternative integrators.** Symplectic / high-order schemes
  (velocity Verlet, leapfrog, Runge–Kutta-Nyström) exposed via input
  selection. Benchmark on H\ :sub:`3` / N\ :sub:`3` / O\ :sub:`3` so
  users can pick the cheapest method that satisfies their accuracy
  needs.
- **Gaussian binning.** Implement the Bonnet-Rayez Gaussian binning
  correction for final-state quantisation; expose as an input option.
  Expected to reduce zero-point-leakage artefacts in low-energy
  collisions.
- **Non-Boltzmann initial distributions.** Support for user-supplied
  initial ro-vibrational distributions (beyond ``Boltzmann`` and
  ``Uniform``) — useful for plasma and laser-excited kinetics.


Longer-term directions
----------------------

- **Nonadiabatic coupling beyond FSSH.** Global-flux surface hopping,
  decoherence-induced surface hopping, coupled-trajectory methods.
  Higher accuracy at the cost of more bookkeeping per trajectory.
- **5-atom systems.** Currently unsupported in CoarseAIR; the work
  is tracked in :doc:`adding_a_system`. First candidate: HN\ :sub:`4`
  / N\ :sub:`4`\ H kinetics for NH\ :sub:`3`-doped air chemistry.
- **Transformer / equivariant NN PES integration.** Architecturally
  straightforward via the existing ``PES_Type`` subclass pattern;
  training and validation are the main effort.
- **Direct coupling to dmsAIR / PLATO.** Currently
  CoarseAIR output → PyCoarseAIR HDF5 → dmsAIR/PLATO. A direct,
  in-memory coupling (via a shared library) would reduce the round-trip
  cost for coupled-multiscale workflows.


How to contribute
-----------------

1. Open a GitHub issue on
   https://github.com/HERMES-AEROSPACE/CoarseAirQCT/issues
   describing the change. This matters especially for anything that
   touches the factory-registered classes or the input-parser schema.
2. Fork / branch off ``Clement-Adiabatic``.
3. Follow the patterns laid out in :doc:`adding_a_system` — subclass
   + factory entry + database assets + input template + regression
   test + documentation page.
4. Run ``make html`` on ``docs/online-documentation`` with ``-W`` to
   verify your documentation edits build cleanly.
5. Submit a pull request against ``Clement-Adiabatic``.

The contribution will be auto-deployed to the documentation site upon
merge via the CI pipeline configured in
``.github/workflows/docs.yml``.
