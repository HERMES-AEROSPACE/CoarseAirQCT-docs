N3 System

The N₃ system is a three-body arrangement of three indistinguishable nitrogen atoms. It is the main system for air-chemistry N2 dissociation studies. Input files live under input/N3/.

Arrangement

  • Atoms: N, N, N (all identical).

  • Collision entry: N + N2.

Kinetic processes

  • Inelastic scattering — \(\mathrm{N}_{2}(v, J) + \mathrm{N} \to \mathrm{N}_{2}(v', J') + \mathrm{N}\).

  • Reactive exchange (identical-atom) — summed into a single channel in indistinguishable mode.

  • Dissociation / recombination — \(\mathrm{N}_{2} + \mathrm{N} \leftrightarrow 3\,\mathrm{N}\). This is the primary dissociation channel of molecular nitrogen in the stratosphere and re-entry shock layer.

Multi-surface structure

N3 has multiple low-lying electronic surfaces (singlet, triplet, quintet) that all contribute to dissociation at high \(T\). The NASA and UMN surfaces each cover several; trajectories are distributed across them with degeneracy weights at initialisation. See Sampling.

PES models

Available:

  • NASA — NASA Ames analytic surface.

  • UMN — University of Minnesota surface (MEG expansion).

  • UMN_MBPIPMEG — PIP-augmented variant.

Diatomic partners: N2_UMN_ForN3, N2_UMN_PIPMEG_ForN3 (matching the full PES).

Typical use

N + N2 dissociation is the central rate in hypersonic air chemistry. CoarseAIR state-to-state N3 rates are used as inputs to master-equation dissociation studies (e.g. in dmsAIR) and provide the reference dataset for coarse-grained model reduction.

Complete input file walkthrough

The working CoarseAIR.inp from input/N3/NASA/input/ illustrates every major block you will need in a real run. The excerpts below are the file in full order (comments trimmed for brevity); see Input File Reference for the meaning of each line.

The code blocks below reproduce the actual template structure — including the ##### banner separators and the indentation convention — so each block can be dropped verbatim into a hand- written CoarseAIR.inp.

Header and pipeline flags:

###########################################################################################
######                                                                               ######
######                         Input File for QCT Code                               ######
######                                                                               ######
###########################################################################################

   Generating Energy Levels?        = no      ! reuse pre-computed levels
   Preprocessing Energy Levels List? = yes    ! bin them into StS or groups
   Running Trajectories?             = yes    ! integrate QCT ensemble
   Postprocessing Trajectories?      = no     ! done in a separate pass

     Molecule 1 = N2

     Atom 1 = N
       Mass [a.u.] of Atom 1 = 25526.04298d0
     Atom 2 = N

     Atom 3 = N
       Mass [a.u.] of Atom 3 = 25526.04298d0

Running the example

cp -r coarseair/input/N3/NASA/input Run_N3/
cp    coarseair/scripts/launching/CoarseAIR.sh Run_N3/
cd    Run_N3/
# Edit input/CoarseAIR.inp to set the absolute Database Path.
COARSEAIR_UPDATE; COARSEAIR_release
bash ./CoarseAIR.sh
# Second pass for rates:
#   Running Trajectories?         = no
#   Postprocessing Trajectories? = yes
bash ./CoarseAIR.sh

The rate output lands under Text/N3/Rates/T_10000_10000/Proc_*.csv. See Post-processing for the format and Getting Started for a broader tutorial.
