PES Formats

CoarseAIR separates the PES interface from the PES implementation: the trajectory integrator only ever calls a small set of virtual methods on an abstract PES_Type object, while the concrete sub-class handles the actual functional form. The design allows 40+ surface implementations — spanning analytic many-body expansions, neural networks, Gaussian processes, and Bayesian ensembles — to be swapped in from the input file without any rebuild.

This chapter describes the PES abstract base class, the factory mechanism, the functional families currently supported, and the multi-surface infrastructure that enables degeneracy-weighted branching across electronic surfaces.

The PES abstract base class

Every surface implements PES_Type (src/PESs/PES_Class.F90), whose deferred methods are:

  • Compute_0d(Q, V, dVdQ, Rpi) — one-point evaluation returning the scalar potential \(V\), its gradient \(\partial V / \partial Q\), and the inverse inter-atomic distances \(R_{pi}^{-1}\) used by the bond-distance analysis.

  • Potential_0d(Q) — potential only, for diagnostics and plotting.

  • TriatPotential_0d / DiatPotential_0d — many-body / two-body decomposition (where the surface is written as \(V = V_{\text{diat}} + V_{\text{triat}}\) or analogous for 4-atom systems).

  • TransToCart_3Atoms / TransToCart_4Atoms — coordinate transformation helpers.

A PES object also owns an array of DiatomicPotential_Type objects (one per atom pair) for the two-body contribution — these are the second-order asymptotic limits the full PES must recover as one atom separates.

Factory mechanism

Selection of the concrete PES at run time is handled by PES_Factory_Class (src/PESs/PES_Factory_Class.F90). The factory keys on system name + model string:

System: O3
PES Model: UMN_13A2    -> constructs an O3_UMN_PES_Type for the 3A''
                          surface with the UMN parametrisation
PES Model: NN_11A1     -> constructs an O3_NN_PES_Type for the 1A'
                          surface with the NN parametrisation
System: CNO
PES Model: UCI_2A1     -> constructs a CNO_UCI_2A1_PES_Type

The factory is also responsible for instantiating the pair-wise DiatomicPotential objects consistent with each PES (e.g. the O3 UMN surface asymptotes to the UMN O2 diatomic potential).

Functional families currently supported

Analytic many-body expansions — the majority of UMN, Basel, NASA, Varandas, and Breno surfaces fall in this class. The surface is written as

\[V(\mathbf{Q}) \;=\; \sum_{i<j} V_{ij}^{(2)}(R_{ij}) \;+\; V^{(3)}\!\left(\{R_{ij}\}\right),\]

where \(V^{(3)}\) is a polynomial in mixed-exponential-Gaussian (MEG) or permutation-invariant polynomial (PIP) basis functions. UMN surfaces for N3, O3, and O4 use 56–200+ expansion coefficients.

Neural-network surfaces — feed-forward networks trained on ab-initio points:

  • NN_PES_Type (generic), O3_NN_PES_Type, etc.

  • Architecture: bond-order input features → two hidden layers with \(\tanh\) activation → scalar output.

  • Weights W1, W2, W3 and biases b1, b2, b3 are read from the database at initialisation; Scaling_MEAN / Scaling_SD normalise the input features.

  • Gradients are back-propagated analytically (no finite differences), so Compute_0d remains exact.

Permutation-invariant NN (PIP-NN) — a PIP layer is prepended to the NN so the surface is by construction symmetric under atom permutations. Used by the N4 UMN_PIPNN and similar surfaces.

Gaussian process (GP) surfaces — kernel-based non-parametric regression. Available for O3 in three noise-level variants. Gradients are computed by differentiating the kernel. Slower per-call than NN but with native uncertainty estimates.

Bayesian neural network (BNN) ensembles — O3 BNN surfaces carry 50–300 sampled weight configurations. The mean surface is used for dynamics; the spread across the ensemble provides uncertainty estimates used by the MATLAB uncertainty-quantification post-processing pipeline (extra/MCoarseAIR/).

LEPS surfaces — London-Eyring-Polanyi-Sato form, retained for historical benchmarks on H3 and O3.

Dummy / non-reactive surfaces — H2–He and CO–Ar use inert-atom “dummy” diatomic potentials (constant or purely repulsive at short range) so the non-reactive atom contributes only to kinetic energy exchange.

Diatomic pair potentials

DiatomicPotential_Type (src/DiatomicPotentials/DiatomicPotential_Class.F90) provides:

  • Compute_Vd_dVd(R) — potential and its first derivative (elemental).

  • Period(v, J) — vibrational period for a given level, used during level-generation and final-state quantisation.

  • ActionIntegral(v, J) / ActionIntegralGamma — WKB action integrals \(\oint p\, dq\) underlying semi-classical quantisation.

  • TurningPoint, FindMinimum, FindMaximum — critical-point location on the effective potential (needed by the arrangement classifier, see Arrangement Classification).

Supported functional forms include Morse, modified-Morse, Schwenke, Schwenke-1988, BMKP (H2), LeRoy (N2), Varandas (O2), and dozens of group-specific parametrisations (UMN, Basel, UCI, UIUC, GUO, NASA, Breno). Each chemical system’s input file references the diatomic potential model consistent with its many-body PES.

Multi-surface handling

CoarseAIR supports running trajectories on multiple electronic surfaces in the same batch. Key data structures (src/Collision/Collision_Class.F90 and src/Collision/Trajectories/Trajectories_Class.F90):

  • Collision%PESsContainer(1:NPESs) — polymorphic array holding each surface object. All surfaces are instantiated at initialisation time.

  • Trajectories%iPES(1:NTraj) — integer mapping from trajectory index to surface index. Each trajectory evaluates V on its own surface.

  • Trajectories%iTrajPES(1:NPESs) — counter of how many trajectories are currently on each surface (for statistics / reporting).

The per-trajectory PES call in HamiltonODESystem is:

! The crux of multi-surface dispatch (simplified):
do iTraj = 1, NTraj
   call Collision%PESsContainer( iPES(iTraj) )%PES% &
        Compute( Q(:,iTraj), V(iTraj), dVdQ(:,iTraj), Rpi(:,iTraj) )
end do

iPES(iTraj) is set at initialisation by SetPESIndx() (with the degeneracy-weighted distribution described in Sampling) and is never modified during integration — each trajectory stays on its assigned surface until termination. The post-processing pipeline reads iPES per trajectory so it can resolve final-state observables by electronic surface.

Adding a new PES

To wire in a new surface:

  1. Create a subclass of PES_Type under src/PESs/, implementing Compute_0d (and optionally the potential-only methods). Place pair-potential assignments in Initialize_PES.

  2. Register the new model name in PES_Factory_Class so the factory can dispatch from the input string.

  3. If the surface has new parameter files, place them under dtb/Systems/<System>/<Model>/.

  4. Rebuild CoarseAIR (COARSEAIR_INSTALL) — the new surface is now selectable from any input file for that system.

A step-by-step walkthrough with a worked example is in Adding a New Collision System.

Functional-form details

Mixed Exponential Gaussian (MEG) basis. UMN surfaces for O3, O4, N3, N4 write the many-body contribution \(V^{(N)}\) as a polynomial in the Morse-like variables

\[y_{ij} \;=\; e^{-\alpha (R_{ij} - R_{ij}^{\text{ref}})}\, e^{-(R_{ij} - R_{ij}^{\text{ref}})^{2}/\beta^{2}},\]

with the exponent producing the correct long-range decay and the Gaussian factor damping the variable at short range. The full many-body potential is a symmetric polynomial in the \(\{y_{ij}\}\) up to some degree (typically 6–9), with 50–200 expansion coefficients fitted to ab-initio data. The MEG variables enter the PES code under src/PESs/ as the a, ab, ra, rb parameters of each X_UMN_PES_Class.F90.

Permutation-Invariant Polynomials (PIP). For a system with \(n\) identical atoms (H3, N4, O4), the PES must be invariant under every permutation of those atoms’ labels. A general polynomial in the inter-atomic distances is not invariant; PIPs are constructed as symmetric polynomial generators — i.e. products of elementary symmetric polynomials in the MEG variables — so that every basis function is already permutation-symmetric. The N4 UMN_PIPNN surface combines a PIP layer with an NN regression on top, preserving the permutation symmetry exactly in the learned representation.

Neural-network forward pass. The generic NN_PES_Type implements a two-hidden-layer feed-forward network

\[V(\mathbf{Q}) \;=\; \mathbf{W}_3\, \tanh\!\bigl(\mathbf{W}_2\, \tanh(\mathbf{W}_1\,\mathbf{x} + \mathbf{b}_1) + \mathbf{b}_2 \bigr) \;+\; \mathbf{b}_3,\]

where \(\mathbf{x}\) is the vector of normalised bond-order input features. The gradient \(\partial V / \partial \mathbf{Q}\) is computed analytically by backpropagation (not by finite differences), so Compute_0d returns exact derivatives at machine precision. Weights and biases are read from the dtb/Systems/<Sys>/NN/ directory at initialisation.

Bayesian NN ensembles. The BNN variants stack up to 300 independent NN samples drawn from the posterior over weights (trained offline by the HERMES uncertainty-quantification tooling). The mean surface is used for trajectory propagation; the spread across the ensemble is the per-point PES uncertainty, consumed by the MATLAB UQ pipeline under extra/MCoarseAIR/. The runtime cost scales linearly in the number of samples, so production BNN runs typically limit the ensemble size to 50.

Gaussian process kernels. The GP_PES_Type evaluates

\[V(\mathbf{Q}) \;=\; \sum_{i=1}^{N_{\text{train}}} \alpha_i\,k(\mathbf{Q}, \mathbf{Q}_i),\]

with \(k\) a squared-exponential or Matérn-5/2 kernel over the inter-atomic distances. The weights \(\alpha_i\) are pre-computed from the training set so runtime inference is a simple weighted sum; the gradient is the kernel derivative, also available in closed form. The three O3 GP variants shipped in dtb/Systems/O3/GP/ differ in the assumed ab-initio noise level and the kernel length-scale.

Dummy (non-reactive) diatomic potentials. For CO–Ar and H2–He the “extra” atom-pair potentials are set to a purely-repulsive short-range form that never supports a bound state, so the classifier can never assign an AB-fragment that includes the inert atom. Used to represent inert collision partners without requiring a full three-body PES fit.
