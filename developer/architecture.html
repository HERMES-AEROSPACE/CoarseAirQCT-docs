Architecture Reference

This page is a developer-oriented map of the CoarseAIR source tree, the main class hierarchies, and the data flow from input parsing to rate-coefficient output. It is intended to help contributors find where to make a change and understand how their change propagates through the pipeline.

For user-facing details of the same pipeline, see the Quasi-Classical Trajectories chapter onwards.

Design patterns

Four patterns used pervasively across the codebase.

Abstract base classes with deferred (virtual) procedures. Every major physics component has an abstract type that declares a small set of deferred methods; concrete subclasses implement them. The integration engine interacts only with the abstract interfaces, so new implementations drop in without changes to the dynamics code.

Factory pattern. Seven factory classes construct concrete objects at runtime based on string keys from the input file:

Factory

Purpose

PES_Factory_Class

Construct PES objects from (system, model) keys.

DiatomicPotential_Factory_Class

Construct diatomic potentials from species + model keys.

Molecule_Factory_Class

Construct molecule objects.

System_Factory_Class

Construct Nb3 or Nb4 system topology.

Processes_Factory_Class

Construct 3-atom or 4-atom process handlers.

Degeneracy_Factory_Class

Construct species-specific degeneracy calculators.

BinsContainer_Factory_Class

Construct binning-strategy objects.

Container pattern. LevelsContainer, PESsContainer, MoleculesContainer, and BinsContainer wrap polymorphic objects so they can be stored in Fortran arrays. Required because Fortran doesn’t natively allow arrays of abstract types.

Modular separation. Physics (PES, levels, processes) and numerics (ODE solver, trajectories, integrator) are independent. They interact only through the Collision_Class orchestrator.

Core data structures

Collision_Type (src/Collision/Collision_Class.F90) — central orchestrator. Owns:

  • NAtoms, NPairs, NPESs.

  • Atoms(:), Species(:), Pairs(:).

  • MoleculesContainer(:) (polymorphic molecule wrappers).

  • PESsContainer(:) (one polymorphic PES per electronic surface).

  • Transformation, ImpactParameter.

  • Methods: Initialize, SetInitialConditions, Hamiltonian, Compute_PES.

Trajectory_Type — single trajectory.

  • PaQ(NEqtTot) — phase-space vector (momenta first, coordinates second).

  • Rpi(NPairs) — inverse pair distances.

  • H — Hamiltonian (conservation diagnostic).

  • dt, t, tmax, RpiMax.

Trajectories_Class — vectorised batch.

  • PaQ(NEqtTot, NTraj).

  • iPES(NTraj) — per-trajectory electronic-surface index.

  • iTrajPES(NPESs) — per-surface counter.

  • b(NTraj), H(NTraj), t(NTraj).

Level_Type — one rovibrational state.

  • vqn, jqn, eint, g.

  • rmin, rmax, ri, ro, tau.

  • Q, QInit, To_Bin.

Process_Type — one collision outcome.

  • ProcType, ExcType (elastic / inelastic / exchange / dissociation).

  • To_Pair(:), To_Species(:), To_Level(:).

  • Temperature(:) — array of rates vs T.

  • ArrCoeffs(3) — Arrhenius (A, n, Ea).

Data flow

CoarseAIR.inp
     │  Input%Initialize()  (src/Input_Class.F90)
     ▼
Collision%Initialize() ──►  Atoms, Species, Pairs
     │                       │
     │  Factory dispatch     ├─► PESsContainer (one per electronic surface)
     │                       ├─► DiatomicPotentials (one per atom pair)
     │                       ├─► Levels (read or computed)
     │                       ├─► Bins (via BinsContainer_Factory)
     │                       └─► Processes (via Processes_Factory)
     ▼
RunTrajectories
     │
     │ Loop over mini-batches:
     │    Collision%SetInitialConditions()
     │    ODE%Integrate_Trajectories()
     │       → ODE%Integrate_ODE_1d()  (Bulirsch-Stoer)
     │          → mmidv()              (modified midpoint)
     │             → HamiltonODESystem()
     │                → Collision%Compute_PES(iPES, Q, dVdQ, V, Rpi)
     │    Collision%Hamiltonian()      (conservation check)
     │    Traj%AssessConvergence()
     │
     ▼  AnalysisTrajPoints → final arrangement, v', J', process type
trajectories.csv
     │
     ▼  TrajectoriesStats
statistics.csv (cross-sections)
     │
     ▼  PostTrajectories (Maxwellian integration)
Proc*.csv (rate coefficients)

Key entry points for extension

  • Add a new PES — subclass PES_Type, register in PES_Factory_Class. See Adding a New Collision System.

  • Add a new integrator — add a sibling to src/Collision/ODE/ODE_Solver.F90, plumb through Integrator_Class.

  • Add a new binning strategy — subclass BinsContainer_Type under src/BinnedMolecules/, register in BinsContainer_Factory_Class.

  • Add a new collision arrangement type (e.g., 5-atom in future) — subclass System_Type, Processes_Type, add a new InitialStates module, and extend the PaQ packing.

  • Add a new PES format (e.g., Transformer-based ML PES) — subclass PES_Type and supply Compute_0d with analytic gradients.

The CoarseAIR+ surface-hopping extension required none of these — it plugged into the existing multi-PES infrastructure and only needed a hopping-decision callback added to the integration loop. See the project_coarseair_plus memory and the CoarseAIR+ documentation.
